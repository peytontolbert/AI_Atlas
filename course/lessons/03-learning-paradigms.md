# Lesson 03: Learning Paradigms

## Goal

Separate the major ways systems learn from data, feedback, interaction, and preferences.

## Review Nodes

`supervised_learning`, `unsupervised_learning`, `self_supervised_learning`, `generative_learning`, `reinforcement_learning`, `model_based_rl`, `preference_learning`, `curriculum_learning`, `meta_learning`, `evolutionary_population_learning`

## Key Ideas

- Supervised learning maps inputs to labeled targets.
- Self-supervised learning creates training signals from the data itself.
- Generative learning models how data can be produced.
- Reinforcement learning optimizes behavior through reward over interaction.
- Preference learning learns from comparative judgments rather than single correct answers.

## Review Checklist

- Can you classify a training setup by paradigm?
- Can you distinguish labels, rewards, and preferences?
- Can you explain why self-supervised pretraining scales well?
- Can you name one failure mode unique to reinforcement learning?
- Can you explain why preference data is useful for assistant behavior?

## Exercise

Choose one product-like AI system, such as a coding assistant or image generator. Identify which paradigms likely appear in its lifecycle:

- Pretraining.
- Task adaptation.
- Feedback collection.
- Evaluation.
- Runtime improvement.

## Deliverable

A lifecycle map connecting paradigms to system stages.

## Common Failure Mode

Using paradigm names as labels without identifying the actual source of the learning signal.
