# Lesson 12: Systems and Applications

## Goal

Combine the atlas layers into complete AI systems with explicit dependencies, evidence, and limits.

## Review Nodes

`multimodal_assistants`, `computer_use_agents`, `code_assistants`, `knowledge_graph_systems`, `scientific_digital_twins`, `autonomous_research`, `theorem_math_systems`, `robotics_embodied_agents`, `recommendation_systems`, `verified_agentic_systems`, `ai_governance_systems`

## Key Ideas

- AI applications are compositions of representations, objectives, models, tools, memory, evals, and infrastructure.
- End-to-end behavior is shaped by every upstream dependency.
- The strongest systems make evidence, uncertainty, provenance, and failure boundaries inspectable.
- A system is not complete until it has evaluation and operational feedback.

## Review Checklist

- Can you trace an application back to at least five atlas layers?
- Can you identify which component owns each failure mode?
- Can you define the evidence required for trust?
- Can you explain the operational boundary of the system?
- Can you state what the system should refuse, escalate, or hand off?

## Exercise

Choose one end-to-end AI system and map it across the atlas:

- Data representation.
- Learning paradigm.
- Objective.
- Model family.
- Retrieval or memory.
- Agent or tool loop.
- Evaluation and safety.
- Serving infrastructure.

## Deliverable

A system dependency map and one-page implementation brief.

## Common Failure Mode

Judging an AI application only by model choice instead of the full dependency chain.
